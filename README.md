## 🚀 Featured Project

### 🧠 [TestCraft QA Agent](https://github.com/Anshul101010/TestCraft-QA-Agent)

> A Claude-powered Azure DevOps automation agent that converts Acceptance Criteria into structured, reviewable test cases and writes them directly to ADO — with full coverage enforcement and zero blind writes.

| Capability | Detail |
|---|---|
| **Coverage rules** | Positive · Negative · Edge Case · Validation — one TC per AC point minimum |
| **Duplicate detection** | Fetches last 15 module TCs before generating anything new |
| **Approval gate** | Nothing writes to ADO without explicit user confirmation |
| **Web TC format** | Playwright-executable steps + auto-generated Locator Registry |
| **Mobile TC format** | Tap-based vocabulary, no locators |
| **AC input sources** | Paste · ADO User Story ID · Excel / PDF / Image upload |
| **Confidence scoring** | Coverage/5 and Executability/5 scored independently per TC |
| **Update mode** | Edit fields or regenerate steps on existing TCs |

## 🧠 How I Think About QA

> *"A test case that can't be executed reliably is just documentation debt. A test suite without negative coverage is a false safety net."*

Good QA engineering is about **coverage logic, not just test count** — knowing which scenarios to derive from an AC point, detecting what's already been tested, and building systems that enforce quality automatically rather than depending on individual vigilance.
